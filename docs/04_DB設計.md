# 04. DB設計

DB: PostgreSQL (Supabase) / ORM: Prisma。実体は `prisma/schema.prisma`。

## ER概要

```
users 1─n note_articles 1─n article_sections
users 1─n article_ideas      note_articles 1─n approval_requests
users 1─1 settings           note_articles 1─n revenue_logs
users 1─n ai_generation_logs（articleId は任意参照）
```

## 設計上のポイント

- **R2対応**: `status` の遷移はAPI層の遷移ガード（`lib/workflow.ts`）で強制。approved / posted への遷移はユーザー操作のエンドポイントのみが実行。
- **将来連携**: `note_url` / `note_post_id` / `posted_at` を予約。API連携実装時はこのフィールドに書き込む。
- **無料/有料切り分け**: セクション単位の `is_paid` で表現し、プレビューで境界を明示。
