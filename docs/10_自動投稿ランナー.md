# 10. note自動投稿ランナー

- 追加日: 2026-07-07
- **重要ルールの変更記録**: 当初要件のR1「AIが勝手に投稿しない」/R3「規約に反する自動操作をしない」について、ユーザーがリスク（noteの利用規約抵触の可能性・アカウント制限リスク・UI変更による動作停止）の説明を受けた上で「公開まで完全自動」を選択した（2026-07-07）。ただし以下のセーフガードは維持する。

## 維持されるセーフガード

| セーフガード | 実装 |
|---|---|
| **承認していない記事は絶対に投稿されない** | ランナーの対象は `status='approved'` かつ `scheduled_at <= now()` のみ。承認・スケジュール設定はどちらもユーザーのボタン操作 |
| 未確認情報が残る記事は投稿されない | `has_unverified_claims = false` 条件 |
| **価格設定は必ずユーザー**（R2維持） | 有料記事は**下書き投入まで**の自動化。価格・有料ライン・公開クリックはnote上で手動 |
| 投稿頻度制限 | 1時間3件上限＋投稿間隔90秒をランナー側でも強制 |
| パスワード非保存 | 初回のみブラウザで手動ログイン。セッションは `.note-profile/`（gitignore済み・ローカルのみ）に保持 |

## 仕組み

- ローカルPCで動くPlaywrightスクリプト（`scripts/note-poster.mjs`）。システムのMicrosoft Edgeを自動操縦する
- Vercel上では動かない（ブラウザセッションが必要なため）。**投稿予定時刻にPCでランナーが起動している必要がある**
- 無料記事: タイトル・本文・ハッシュタグを入力し公開まで実行 → `posted` に更新、URLを記録
- 有料記事: タイトル・本文を下書き投入 → `copied` に更新（ユーザーが価格設定して公開後、アプリで「投稿済み」を押す）
- 失敗時: スクリーンショットを `poster-logs/` に保存し、記事はスキップ（statusは変更しない。次回再試行）

## 使い方

```bash
npm run poster:test   # お試し: 期限が来た記事を「下書き投入まで」で1回実行（公開しない）
npm run poster:once   # 期限が来た記事を処理して終了（無料記事は公開まで）
npm run poster        # 常駐監視（60秒ごとにチェック。投稿予定の時間帯はこれを起動しておく）
```

1. 初回はブラウザ（Edge）が開くので、note にログインする（5分以内）
2. 以降はログイン状態が再利用される
3. **初回は必ず `poster:test` → note上で下書きの仕上がりを確認**してから本運用へ

## 既知の制約

- noteのUIはHTML構造が予告なく変わるため、動かなくなったらセレクタの調整が必要（`poster-logs/` のスクリーンショットが手がかり）
- 会社ネットワークではNeonへの接続に `ca-bundle.pem` が必要（npmスクリプトに設定済み）
- 短時間に大量投稿するとnote側の不正検知に触れる可能性が上がるため、上限は緩めないこと
