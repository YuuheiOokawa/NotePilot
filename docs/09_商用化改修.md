# 09. 商用化改修（ボタンを押すだけの運用へ）

- 追加日: 2026-07-07
- 方針: **ユーザーの操作を「ボタン押下」だけに減らす。** ただし承認・投稿は必ずユーザーのボタン操作（自動化しない）。

## 追加・変更した機能

### 1. 生成と品質チェックの自動連結
- 記事生成APIが**生成後すぐ品質チェックまで自動実行**（`lib/articleService.ts` + `lib/quality.ts`）
- ユーザーは「生成ボタン1回」で「下書き＋チェック済み（問題なければ投稿準備OK）」まで進む
- チェック失敗時も記事は保存され、後から手動チェック可能

### 2. シリーズ一括生成
- `POST /api/theme-groups/[id]/generate-next`: 次の未生成項目を1件生成（チェック込み）
- シリーズ詳細の「⚡ 残りN本をまとめて生成」ボタンが残り0になるまで自動で繰り返し呼ぶ
- 1リクエスト=1記事のため、Vercelの関数タイムアウトに抵触しない。途中失敗しても再度ボタンで続きから再開できる

### 3. 1タップ承認
- 記事詳細: draftかつ投稿準備OKの記事に「✅ 承認まで進める」ボタン（確認に出す→承認を一括実行）
- 承認はこのボタン＝ユーザー操作なので承認制ルールは維持

### 4. 投稿スケジュール自動割当
- `POST /api/schedule/auto-assign`: 承認済み・予定未設定の記事に、翌日以降の空き時間帯（**JST 9〜21時の主要時間帯、1時間1件で分散**、1時間3件上限は遵守）を自動割当
- 投稿キュー画面の「⚡ 自動割当」ボタン1つ

### 5. ホームの「次にやること」ガイド
- 記事の状態から次の一手を判定し、優先度順にボタン表示（承認待ち→承認へ / チェックOK下書き→承認へ / 承認済み→コピーへ / 未スケジュール→自動割当 / コピー済み→投稿記録）
- 記事一覧はURLクエリ（`?status=draft`等）で直接絞り込み表示可能に

### 6. UI/UX の商用化
- **トースト通知**（`components/Toast.tsx`）: alert() を全廃し、成功/エラーを非ブロッキング表示
- 生成系ボタンの多重押下防止・進捗表示（一括生成は「x/y本完了」）

### 7. 信頼性・環境差异の修正
- **hourKey をJST固定化**: ローカル（JST）とVercel（UTC）が同じDBを共有しても時間バケツ集計がずれないよう、サーバーTZ非依存に変更
- 全AI系ルートに `maxDuration = 60` を設定（実AI接続時のVercelタイムアウト対策）

## 変わらないもの（重要ルール）

- 承認（approved）・投稿記録（posted）は必ずユーザーのボタン操作
- 未確認情報が残る記事は承認不可（自動チェックで unverified が出た場合、ユーザーが確認するまで止まる）
- noteへの自動投稿はしない。スケジュールは管理用の目安

## ボタンだけ運用フロー（最短）

1. 「シリーズ」→ テーマ入力 → **AI企画ボタン** → **保存ボタン**
2. シリーズ詳細 → **⚡一括生成ボタン**（全記事が生成＋チェック済みに）
3. ホームの「次にやること」→ 各記事で **✅承認まで進めるボタン**（要確認情報があれば確認ボタンで解消）
4. 投稿キュー → **⚡自動割当ボタン**（投稿予定が組まれる）
5. 予定時刻になったら記事の **コピーボタン** → **noteを開くボタン** → 貼り付け → **投稿済みボタン**
