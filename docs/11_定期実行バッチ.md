# 11. 定期実行バッチ

- 追加日: 2026-07-07
- 目的: 「ボタンだけ運用」をさらに進め、日々の定型作業（記事生成・スケジュール割当）を無人化する。**承認と実投稿の判断は引き続きユーザーが行う**（重要ルールR1/R2は維持）。

## 全体像

```
[Vercel Cron 毎日9:00 JST]
  /api/cron/daily
    1. シリーズの未生成記事を自動生成（品質チェックまで）→ 承認待ちに積まれる
    2. 承認済み・予定未設定の記事へ投稿スケジュールを自動割当

[ユーザー]
  ホームの「次にやること」で ✅承認（ここだけ人間）

[ローカルPC 投稿ランナー（任意・定期実行可）]
  scripts/note-poster.mjs
    予定時刻が来た承認済み記事を投稿（無料=公開まで / 有料=下書きまで）
```

## 1. Vercel Cron（サーバー側バッチ）

`vercel.json` に定義済み。デプロイするだけで有効になる。

```json
{ "crons": [{ "path": "/api/cron/daily", "schedule": "0 0 * * *" }] }
```

- スケジュールは **UTC 0:00 = JST 9:00**（Hobbyプランは1日1回まで・実行時刻は多少前後する）
- やること:
  - 未生成のシリーズ記事を最大 `CRON_GENERATE_LIMIT` 件（既定3件）生成＋品質チェック
  - `approved`/`copied` で予定未設定の記事に空き時間帯（JST 9〜21時、1時間1件）を割当
- タイムアウト対策: 生成は40秒で打ち切り、残りは翌日に持ち越す

### 保護（推奨）

Vercelの環境変数に `CRON_SECRET`（ランダムな長い文字列）を設定する。
VercelはCronのリクエストに `Authorization: Bearer <CRON_SECRET>` を自動で付けるため、
第三者が `/api/cron/daily` を直接叩いても401になる。

### 手動実行・動作確認

```bash
curl -H "Authorization: Bearer $CRON_SECRET" https://<デプロイURL>/api/cron/daily
# → {"ok":true,"generated":[...],"generatedCount":2,"assignedCount":3,...}
```

実行履歴は Vercel ダッシュボード > プロジェクト > Cron Jobs / Logs で確認できる。

## 2. 投稿ランナーの定期実行（ローカルPC側・任意）

実投稿はブラウザセッションが必要なためローカルでのみ動く（`docs/10_自動投稿ランナー.md`）。
常駐（`npm run poster`）の代わりに、OSのスケジューラで `poster:once` を定期起動してもよい。

### Windows（タスクスケジューラ）

管理者PowerShellで一度実行（毎時0分に起動する例）:

```powershell
schtasks /Create /TN "NotePoster" /SC HOURLY /ST 09:00 ^
  /TR "cmd /c cd /d C:\path\to\NotePilot && npm run poster:once >> poster-logs\cron.log 2>&1"
```

- 投稿予定はJST 9〜21時に割当されるため、`/SC HOURLY` で9:00〜21:00の間PCが起きていれば拾える
- 削除は `schtasks /Delete /TN "NotePoster"`

### macOS / Linux（cron）

```bash
crontab -e
# 毎日9〜21時の毎時0分に1回実行
0 9-21 * * * cd /path/to/NotePilot && npm run poster:once >> poster-logs/cron.log 2>&1
```

### 注意

- ランナーは `status='approved'` かつ `scheduled_at <= now` の記事だけを処理する。承認していない記事が投稿されることはない
- 初回はログインセッション作成のため手動で `npm run poster:test` を実行しておくこと
- PCがスリープしていた場合、次の起動時に期限超過分をまとめて処理する（1時間3件上限・90秒間隔は維持）

## 設計メモ

- 生成と割当を1つのエンドポイントにまとめたのは、Vercel Hobbyの「Cronは1日1回×2本まで」制限のため
- `autoAssignSchedules()` は画面の「⚡自動割当」ボタンと共通実装（`src/lib/autoAssign.ts`）
- 生成済み記事は従来どおり draft → ユーザー承認フローに乗る。バッチが承認や投稿を行うことはない
